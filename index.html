题目描述

给你一个由 n 个整数组成的数组 nums ，和一个目标值 target 。请你找出并返回满足下述要求的四元组 [nums[a], nums[b], nums[c], nums[d]] （若两个四元组元素一一对应，则认为两个四元组相同）：

  • 0 <= a, b, c, d < n
  • a、b、c 和 d 互不相同
  • nums[a] + nums[b] + nums[c] + nums[d] == target

你可以按 任意顺序 返回答案 。

解题思路

这道题是三数之和的扩展，需要找到四个数之和等于目标值。

暴力枚举时间复杂度 O(N^4)，不可行。

采用排序 + 双指针的方法：

  1. 先对数组排序。

  2. 外层两层循环固定前两个数 i 和 j。

  3. 内层使用双指针 left 和 right，从 j+1 和末尾开始，向中间移动。

  4. 根据当前和与 target 的比较，移动指针。

  5. 去重：跳过重复的元素。

这样时间复杂度 O(N^3)，空间复杂度 O(1)（不计结果空间）。

需要注意溢出，使用 long 类型计算和。

复杂度分析

  • 时间复杂度：O(N^3)，其中 N 是数组长度。外层两层循环 O(N^2)，内层双指针 O(N)。

  • 空间复杂度：O(1)，除了存储结果的空间。

题目描述

给定四个包含整数的数组列表 A , B , C , D ,计算有多少个元组 (i, j, k, l) ，使得 A[i] + B[j] + C[k] + D[l] = 0。

解题思路

这道题如果用暴力枚举四个数组的所有组合，时间复杂度会是 O(N^4)

我们可以使用哈希表来优化。核心思想是将四个数的和转化为两个数的和的组合。

具体步骤：

  1. 使用一个哈希表来统计数组 A 和 B 中所有数对 (i, j) 的和 A[i] + B[j] 出现的次数。

  2. 然后遍历数组 C 和 D，对于每一对 (k, l)，计算 -(C[k] + D[l])，在哈希表中查找这个值出现的次数，加到结果中。

这样，时间复杂度降为 O(N^2)，空间复杂度为 O(N^2)。

代码实现

复杂度分析

  • 时间复杂度：O(N^2)，其中 N 是数组长度。构建哈希表需要 O(N^2)，查询也需要 O(N^2)。

  • 空间复杂度：O(N^2)，哈希表存储了所有 A 和 B 的数对和。

题目描述

给定一个大小为 n 的数组 nums ，返回其中的多数元素。多数元素是指在数组中出现次数 大于 ⌊ n/2 ⌋ 的元素。

你可以假设多数元素总是存在的，并且数组是非空的。

示例 1：

输入：nums = [3,2,3]
输出：3

示例 2：

输入：nums = [2,2,1,1,1,2,2]
输出：2

提示：

  • n == nums.length
  • 1 <= n <= 5 * 10^4
  • -10^9 <= nums[i] <= 10^9

哈希表方法解题思路

我们可以使用一个哈希表来记录数组中每个元素出现的次数。具体步骤如下：

  1. 初始化哈希表：创建一个空的哈希表，用于存储元素及其出现次数。

  2. 遍历数组统计次数：对数组中的每个元素进行遍历，如果哈希表中已经存在该元素，则将其出现次数加1；否则，将该元素添加到哈希表中，并设置出现次数为1。

  3. 查找多数元素：再次遍历哈希表，找到出现次数大于数组长度一半（即 nums.length / 2）的元素。由于题目保证多数元素存在，我们可以直接返回该元素。

代码实现

复杂度分析

  • 时间复杂度：O(n)，需要遍历数组两次（一次统计，一次查找）。
  • 空间复杂度：O(n) 在最坏情况下，所有元素都不同，需要存储n个键值对。

  • 使用一个指针 cnt 来记录不重复元素的个数，初始为1（第一个元素肯定不重复）。

题目描述

题目要求 :
给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出和为目标值 target 的那两个整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案。但是，数组中同一个元素在答案里不能重复出现。

你可以按任意顺序返回答案。

示例

输入： nums = [2,7,11,15], target = 9
输出： [0,1]
解释： 因为 nums[0] + nums[1] == 9 ，我们返回 [0, 1] 。

输入： nums = [3,2,4], target = 6
输出： [1,2]

输入： nums = [3,3], target = 6
输出： [0,1]

解题思路

这个问题可以使用哈希表来高效解决。基本思路是：

  1. 创建一个哈希表，用于存储数组元素及其对应的下标。
  2. 遍历数组，对于每个元素 nums[i]，计算其补数 complement = target - nums[i]。
  3. 检查哈希表中是否已经存在这个补数，如果存在且不是当前元素本身（避免同一个元素使用两次），则返回这两个下标。
  4. 如果不存在，将当前元素及其下标存入哈希表。

这种方法的时间复杂度为 O(n)，比暴力枚举的 O(n²) 更高效。

  • 解题思路 :
    • 因为给你两个按 非递减顺序 排列的整数数组nums1 和 nums2,相当于归并排序中已经排好序的左部分和右部分,所以我先想到的是用归并排序中merge的过程来将两个数组合并

使用归并排序中merge的过程来将两个数组合并

  • 变量定义 :
  • index1 : 表示nums1数组中有值的索引位置
  • index2 : 表示nums2数组的索引位置
  • i : 表示辅助数组nums3的索引位置
  • nums3 : 表示辅助数组,用来存储合并后的结果
  • 思路 :
  • 因为nums1数组中有值的索引位置为0到m-1,所以index1初始值为0,当index1小于m时说明nums1数组中还有值
  • 因为nums2数组的索引位置为0到n-1,所以index2初始值为0,当index2小于n时说明nums2数组中还有值,所以循环条件为 while (index1 < m && index2 < n)
  • 因为nums1数组和nums2数组都是按 非递减顺序 排列的,所以每次比较nums1数组中index1位置的值和nums2数组中index2位置的值,将较小的值添加到辅助数组nums3中,并且将对应的索引位置加1,所以 nums3[i++] = nums1[index1] < nums2[index2] ? nums1[index1++] : nums2[index2++];
  • 当右部分索引越界后,将左部分中剩余的元素全部添加到辅助数组nums3中while (index1 < m) { nums3[i++] = nums1[index1++]; }
  • 当左部分索引越界后,将右部分中剩余的元素全部添加到辅助数组nums3中while (index2 < n) { nums3[i++] = nums2[index2++]; }
  • 将辅助数组nums3中的元素全部添加到nums1数组中for (i = 0; i < m + n; i++) { nums1[i] = nums3[i]; }

    • 如果快指针和慢指针相遇了,说明链表有环,所以需要一个指针从头节点开始,一个指针从相遇节点开始,每次移动一步,直到两个指针相遇了,说明找到了链表开始入环的第一个节点,所以循环条件为 while (index1 != index2)
      • 如果快指针和慢指针没有相遇了,说明链表无环,所以返回null return null;

LeetCode题目

  • 题目要求 : 给你单链表的头节点 head ，请你反转链表，并返回反转后的链表。

一.使用双指针

  • 变量定义 :
  • head: 表示当前位置 (变化: 1 -> 2 -> 3 -> null)
  • pre: 当前位置的上一位置也是当前位置即将指向的下一位置(反转链表的指向) (变化: null-> 1 —> 2 -> 3)
  • next: 当前位置的下一位置也是当前位置即将被指向的上一位置(反转链表的指向) (变化: null -> 2 -> 3-> null)
  • 解题思路 :
  • 因为需要反转链表,所以需要一个pre指针来指向当前节点的上一位置,因为当前节点的下一位置会被改变,所以需要一个next指针来指向当前节点的下一位置
  • 当head不为空时,说明还有节点需要反转,所以循环条件为 while(head!=null)
  • 在循环中,首先将next指针移到当前节点的下一位置 next = head.next;
  • 然后将当前节点的下一位置指向pre指针指向的位置,即反转链表的指向 head.next = pre;
  • 然后将pre指针移到当前节点,pre = head;
  • 最后将head指针移到next指针指向的位置head = next;
  • 当循环结束时,pre指针指向的就是反转后的链表的头节点,所以返回pre指针指向的位置 return pre;

二.使用递归

  • 变量定义 :
  • cur: 表示当前节点
  • pre: 当前位置的上一位置也是当前位置即将指向的下一位置(反转链表的指向)
  • next: 当前位置的下一位置也是当前位置即将被指向的上一位置(反转链表的指向)

LeetCode题目

  • 题目要求 : 给你一个链表的头节点 head 和一个整数 val ，请你删除链表中所有满足 Node.val == val 的节点，并返回 新的头节点 。
  • 思路 : 使用虚拟头节点
  • 变量定义 :
  • dummyHead : 表示虚拟头节点
  • cur : 表示当前节点
  • 解题思路 :
  • 因为要删除链表中所有满足 Node.val == val 的节点,所以需要一个虚拟头节点来处理头节点被删除的情况,所以dummyHead初始值为一个新的节点,且该节点的next指向head ListNode dummyHead = new ListNode(-1,head);
  • 因为需要遍历链表来删除满足条件的节点,所以cur初始值为虚拟头节点 ListNode cur = dummyHead;
  • 当cur的下一个节点不为空时,说明还有节点需要判断,所以循环条件为 while(cur.next!=null)
    • 如果cur的下一个节点的值等于val,说明需要删除该节点,所以将cur的下一个节点指向下一个节点的下一个节点 cur.next = cur.next.next;
    • 如果cur的下一个节点的值不等于val,说明不需要删除该节点,所以将cur指向下一个节点 cur = cur.next;
    • 最后返回虚拟头节点的下一个节点,即新的头节点 return dummyHead.next;

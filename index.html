题目描述

给你一个字符串表达式 s ，请你实现一个基本计算器来计算并返回它的值。

整数除法仅保留整数部分。你可以假设给定的表达式总是有效的。所有中间结果将在 [-2^31, 2^31 - 1] 内。

注意：不允许使用任何将字符串作为数学表达式计算的内置函数，比如 eval()。

解题思路

这道题需要处理加减乘除运算，且乘除优先级高于加减。我们可以使用栈来处理：

  • 遍历字符串，跳过空格。
  • 当遇到数字时，累积数字值。
  • 当遇到运算符或到达末尾时，根据前一个运算符处理当前数字：
    • 如果是 ‘+’ 或 ‘-‘，直接将数字压入栈（减号时压入负数）。
    • 如果是 ‘*’ 或 ‘/‘，从栈顶弹出前一个数字，与当前数字运算后压回栈。
  • 更新运算符为当前字符。
  • 遍历结束后，将栈中所有数字相加即为结果。

这种方法利用栈来处理运算符优先级：乘除立即计算，加减延迟到最后求和。

代码实现

复杂度分析

  • 时间复杂度：O(n)，其中 n 是字符串 s 的长度。我们需要遍历字符串一次。
  • 空间复杂度：O(n)，在最坏情况下（表达式全为加法），栈的空间为 O(n)。

题目描述

给你一个字符串数组 tokens ，表示一个根据逆波兰表示法表示的算术表达式。

请你计算该表达式。返回一个表示表达式值的整数。

注意：

  • 有效的算符为 ‘+’、’-‘、’*’ 和 ‘/‘ 。
  • 每个操作数（运算对象）都可以是一个整数或者另一个表达式。
  • 两个整数之间的除法总是向零截断。
  • 表达式中不含除零运算。
  • 输入是一个根据逆波兰表示法表示的算术表达式。
  • 答案及所有中间计算结果可以用 32 位整数表示。

解题思路

逆波兰表达式（后缀表达式）不需要括号，运算符在操作数之后。我们可以使用栈来求值：

  • 遍历 tokens 数组。
  • 如果遇到操作数，将其转换为整数并压入栈中。
  • 如果遇到运算符，从栈中弹出两个操作数（注意顺序：先弹出的是右操作数，后弹出的是左操作数），进行运算，然后将结果压入栈中。
  • 遍历结束后，栈中只剩下一个元素，即为最终结果。

对于运算符：

  • “+” : 左 + 右
  • “-“ : 左 - 右
  • “*” : 左 * 右
  • “/“ : 左 / 右（整数除法，向零截断）

代码实现

复杂度分析

  • 时间复杂度：O(n)，其中 n 是 tokens 的长度。我们需要遍历整个数组一次，每个元素进行常数时间的操作。
  • 空间复杂度：O(n)，栈的空间在最坏情况下（所有元素都是操作数）为 O(n)。

题目描述

给出由小写字母组成的字符串 s，重复项删除操作会选择两个相邻且相同的字母，并删除它们。

在 s 上反复执行重复项删除操作，直到无法继续删除。

在完成所有重复项删除操作后，返回最终的字符串。答案保证唯一。

解题思路

这道题可以使用栈来解决。我们遍历字符串中的每个字符：

  • 如果栈为空或者当前字符与栈顶字符不同，将当前字符压入栈中。
  • 如果当前字符与栈顶字符相同，弹出栈顶字符（相当于删除这两个相邻的重复字符）。

由于删除操作是反复进行的，而栈的这种操作正好模拟了这个过程：在遇到重复时弹出，等价于删除。

最终，栈中剩余的字符就是结果。

在代码实现中，我们使用一个字符数组来模拟栈，避免频繁的字符串操作，提高效率。

题目描述

给定一个只包括 ‘(‘，’)’，’{‘，’}’，’[‘，’]’ 的字符串 s ，判断字符串是否有效。

有效字符串需满足：

  1. 左括号必须用相同类型的右括号闭合。
  2. 左括号必须以正确的顺序闭合。
  3. 每个右括号都有一个对应的相同类型的左括号。

解题思路

这道题可以使用栈来解决。我们遍历字符串中的每个字符：

  • 如果遇到左括号（’(‘、’{‘ 或 ‘[‘），将其压入栈中。
  • 如果遇到右括号（’)’、’}’ 或 ‘]’），检查栈是否为空。如果为空，返回 false。如果不为空,则弹出栈顶元素，检查是否与当前右括号匹配。如果不匹配，返回 false。
  • 遍历结束后，检查栈是否为空。如果为空，说明所有括号都匹配，返回 true；否则，返回 false。

此外，我们可以先检查字符串长度是否为奇数，如果是，直接返回 false，因为有效的括号字符串长度必须为偶数。

代码实现

复杂度分析

  • 时间复杂度：O(n)，其中 n 是字符串 s 的长度。我们需要遍历字符串一次，每个字符进行常数时间的操作。
  • 空间复杂度：O(n)，在最坏情况下，字符串中所有字符都是左括号，我们需要将它们全部压入栈中，栈的空间复杂度为 O(n)。

问题描述

给定一个字符串 s，你需要将其中的单词顺序反转。单词由非空格字符组成，单词之间由至少一个空格分隔。

返回一个字符串，其中单词顺序反转，并用单个空格连接。注意，输入字符串可能包含前导或尾随空格，或单词之间有多个空格，但输出字符串不应包含前导或尾随空格，且单词之间只有一个空格。

示例：

  • 输入: s = “the sky is blue”
  • 输出: “blue is sky the”

解题思路

解决这个问题，我们采用原地修改字符串的方法，通过两次反转来实现单词顺序的反转。

首先，将整个字符串反转。此时，单词的顺序已经反转，但每个单词的字符顺序也被反转了。

接下来，我们需要修正每个单词的字符顺序，并处理空格。

使用两个指针：快指针 fast 用于遍历原字符串，慢指针 slow 用于构建新字符串。

  • 跳过前导空格：在开始处理之前，使用 fast 指针跳过字符串开头的连续空格，确保我们从第一个非空格字符开始。这是因为输入字符串可能有前导空格，我们不需要将它们复制到结果中。具体来说，使用一个内层循环 while (fast < c.length && c[fast] == ' ') fast++;，如果 fast 到达字符串末尾，则表示整个字符串都是空格，直接跳出外层循环。

  • 当遇到非空格字符时，开始一个单词：记录单词起始位置 index1 = slow，然后将单词字符复制到 slow 位置，同时移动 slow 和 fast。

  • 复制完单词后，反转该单词的字符（从 index1 到 slow-1）。

  • 如果还有空间，添加一个空格到 slow 位置。

重复此过程，直到遍历完整个字符串。

  • 去除末尾的多余空格：在所有单词处理完毕后，由于我们可能在最后一个单词后添加了一个空格，但如果没有更多单词，这个空格就是多余的。因此，使用一个循环 while (slow > 0 && c[slow-1] == ' ') slow--;，从 slow-1 开始向前检查，如果遇到空格，就减少 slow，直到遇到非空格字符或 slow 为 0。这样确保结果字符串没有尾随空格，并且不会删除单词中的空格。

最后，返回从 0 到 slow 的子字符串。

时间复杂度是 O(n)，其中 n 是字符串的长度。具体来说：

  • 整体反转字符串需要 O(n) 时间，因为需要遍历整个字符串进行交换。
  • 遍历字符串并处理单词的过程中，快指针 fast 和慢指针 slow 各遍历字符串一次，时间为 O(n)，在此过程中还包括跳过空格和复制字符的操作。
  • 反转每个单词的时间总和也是 O(n)，因为每个字符在整个过程中最多被反转两次（一次整体反转，一次单词反转），但实际上单词反转只针对单词字符。

因此，总时间复杂度为 O(n)，这意味着算法的运行时间与输入字符串长度成线性关系，非常适合处理长字符串。

空间复杂度是 O(n)，因为我们将输入字符串转换为字符数组以进行原地修改。这个字符数组的大小与输入字符串长度成正比。除了这个数组，我们只使用了常数个额外变量，因此总空间复杂度为 O(n)。

问题描述

给定一个字符串 s 和一个整数 k，你需要对字符串进行如下操作：

  • 从字符串的开头开始，每计数至 2k 个字符，就反转这 2k 字符中的前 k 个字符。
  • 如果剩余字符少于 k 个，则将剩余字符全部反转。
  • 如果剩余字符大于等于 k 个但小于 2k 个，则反转前 k 个字符，其余字符保持原样。

示例：

  • 输入: s = “abcdefg”, k = 2
  • 输出: “bacdfeg”

解题思路

要解决这个问题，我们需要模拟字符串的反转过程，按照每 2k 个字符的块进行处理。

  1. 初始化：将字符串转换为字符数组，因为字符串在 Java 中是不可变的，便于原地修改。

  2. 遍历字符串：使用一个循环，起始索引 i 从 0 开始，每次增加 2*k，以处理每个块。

  3. 确定反转范围：对于每个块，左边界 left = i，右边界 right = min(i + k - 1, s.length() - 1)。这样确保只反转前 k 个字符，如果剩余字符少于 k 个，则反转所有剩余字符。

  4. 反转操作：使用双指针法，从 left 和 right 开始，向中间移动，交换字符，直到 left >= right。

  5. 返回结果：将字符数组转换回字符串并返回。

时间复杂度：O(n)，因为每个字符最多被访问两次（一次遍历，一次反转）。

空间复杂度：O(n)，因为我们将字符串转换为字符数组以进行原地修改。这个字符数组的大小与输入字符串长度成正比。除了这个数组，我们只使用了常数个额外变量，因此总空间复杂度为 O(n)。

题目描述

编写一个函数，其作用是将输入的字符串反转过来。输入字符串以字符数组 s 的形式给出。

不要给另外的数组分配额外的空间，你必须原地修改输入数组、使用 O(1) 的额外空间解决这一问题。

你可以假设数组中的所有字符都是 ASCII 码表中的可打印字符。

示例 1：

输入：s = [“h”,”e”,”l”,”l”,”o”]

输出：[“o”,”l”,”l”,”e”,”h”]

示例 2：

输入：s = [“H”,”a”,”n”,”n”,”a”,”h”]

输出：[“h”,”a”,”n”,”n”,”a”,”H”]

解题思路

这个问题要求原地反转字符串，不能使用额外空间。我们可以使用双指针法：一个指针从字符串开头开始，另一个从结尾开始，交换两个指针指向的字符，然后指针向中间移动，直到相遇。

具体步骤：

  1. 检查输入数组是否为空或长度小于等于1，如果是，直接返回。

  2. 初始化两个指针：i = 0, j = s.length - 1

  3. 当 i < j 时，交换 s[i] 和 s[j]，然后 i++，j–

这种方法的时间复杂度是 O(n)，空间复杂度是 O(1)

题目描述

给你一个由 n 个整数组成的数组 nums ，和一个目标值 target 。请你找出并返回满足下述要求的四元组 [nums[a], nums[b], nums[c], nums[d]] （若两个四元组元素一一对应，则认为两个四元组相同）：

  • 0 <= a, b, c, d < n
  • a、b、c 和 d 互不相同
  • nums[a] + nums[b] + nums[c] + nums[d] == target

你可以按 任意顺序 返回答案 。

解题思路

这道题是三数之和的扩展，需要找到四个数之和等于目标值。

暴力枚举时间复杂度 O(N^4)，不可行。

采用排序 + 双指针的方法：

  1. 先对数组排序。

  2. 外层两层循环固定前两个数 i 和 j。

  3. 内层使用双指针 left 和 right，从 j+1 和末尾开始，向中间移动。

  4. 根据当前和与 target 的比较，移动指针。

  5. 去重：跳过重复的元素。

这样时间复杂度 O(N^3)，空间复杂度 O(1)（不计结果空间）。

需要注意溢出，使用 long 类型计算和。

复杂度分析

  • 时间复杂度：O(N^3)，其中 N 是数组长度。外层两层循环 O(N^2)，内层双指针 O(N)。

  • 空间复杂度：O(1)，除了存储结果的空间。

题目描述

给定四个包含整数的数组列表 A , B , C , D ,计算有多少个元组 (i, j, k, l) ，使得 A[i] + B[j] + C[k] + D[l] = 0。

解题思路

这道题如果用暴力枚举四个数组的所有组合，时间复杂度会是 O(N^4)

我们可以使用哈希表来优化。核心思想是将四个数的和转化为两个数的和的组合。

具体步骤：

  1. 使用一个哈希表来统计数组 A 和 B 中所有数对 (i, j) 的和 A[i] + B[j] 出现的次数。

  2. 然后遍历数组 C 和 D，对于每一对 (k, l)，计算 -(C[k] + D[l])，在哈希表中查找这个值出现的次数，加到结果中。

这样，时间复杂度降为 O(N^2)，空间复杂度为 O(N^2)。

代码实现

复杂度分析

  • 时间复杂度：O(N^2)，其中 N 是数组长度。构建哈希表需要 O(N^2)，查询也需要 O(N^2)。

  • 空间复杂度：O(N^2)，哈希表存储了所有 A 和 B 的数对和。

题目描述

给定一个大小为 n 的数组 nums ，返回其中的多数元素。多数元素是指在数组中出现次数 大于 ⌊ n/2 ⌋ 的元素。

你可以假设多数元素总是存在的，并且数组是非空的。

示例 1：

输入：nums = [3,2,3]
输出：3

示例 2：

输入：nums = [2,2,1,1,1,2,2]
输出：2

提示：

  • n == nums.length
  • 1 <= n <= 5 * 10^4
  • -10^9 <= nums[i] <= 10^9

哈希表方法解题思路

我们可以使用一个哈希表来记录数组中每个元素出现的次数。具体步骤如下：

  1. 初始化哈希表：创建一个空的哈希表，用于存储元素及其出现次数。

  2. 遍历数组统计次数：对数组中的每个元素进行遍历，如果哈希表中已经存在该元素，则将其出现次数加1；否则，将该元素添加到哈希表中，并设置出现次数为1。

  3. 查找多数元素：再次遍历哈希表，找到出现次数大于数组长度一半（即 nums.length / 2）的元素。由于题目保证多数元素存在，我们可以直接返回该元素。

代码实现

复杂度分析

  • 时间复杂度：O(n)，需要遍历数组两次（一次统计，一次查找）。
  • 空间复杂度：O(n) 在最坏情况下，所有元素都不同，需要存储n个键值对。
